## 1. Photography Production Brief

### 1.1 Overview

All photography must feature actual Whites-printed products in authentic Deira business contexts. No stock photography. Images must convey professionalism, quality, and the "Color Your Business" ethos. Warm lighting, shallow depth of field, navy/white branded accents throughout.

### 1.2 Equipment Recommendations for Photographer

- Camera: Full-frame DSLR or mirrorless (Canon EOS R5 / Sony A7IV or better)
- Lens: 24-70mm f/2.8 (versatile), 100mm f/2.8 macro (product details), 50mm f/1.8 (ambient)
- Lighting: Two softbox strobes + portable LED panel (Rotalux style)
- Tripod: Manfrotto or equivalent for consistent hero compositions
- Backdrop: Navy muslin, white seamless paper, branded backdrop with Whites logo
- Calibration: X-Rite ColorChecker Passport for accurate CMYK reproduction

### 1.4 Styling Guidelines

**Lighting:**
- Hero and about shots: warm natural light (golden hour quality) supplemented by softboxes
- Product shots: diffused strobes at 45° with fill card — no harsh shadows
- Technology shots: ambient with LED panel for detail, avoid reflections on glossy panels

**Background:**
- Navy (#0A1F3F) and white surfaces as primary backgrounds
- Real wood desks (light oak, walnut) for lifestyle warmth
- Avoid busy patterns — keep focus on the printed product
- Whites-branded desk mats or coasters as subtle props

**Props:**
- Minimal — a quality pen, a coffee cup, a smartphone, reading glasses
- No food, no personal photos, no clutter
- Select props reinforce "professional business" context

**Models:**
- No identifiable faces unless client provides written release
- Hands only for product demonstration (clean nails, no jewelry)
- Milan and Razak may appear from behind or engaged in work (side/back profile acceptable)

### 1.5 File Naming Convention

```
Frontend/public/images/{category}/{description}-{index}.webp
Frontend/public/images/{category}/{description}-{index}.jpg
```

Use kebab-case descriptions. Index only for multiple variants. Always provide WebP primary + JPEG fallback.

## 3. Placeholder & Fallback Strategy

### 3.1 Placeholder Types

All placeholders stored in `Frontend/public/images/placeholders/`.

### 3.2 SVG Geometric Placeholders

For each image slot, a geometric SVG placeholder is used until the real photograph is available. These follow a consistent navy/white/cyan geometric abstract style.

**Pattern library (`Frontend/public/images/placeholders/patterns/`):**

| File | Description | Colors |
|------|-------------|--------|
| `pattern-dots.svg` | Repeating dot grid | #0A1F3F bg, rgba(255,255,255,0.05) dots |
| `pattern-triangles.svg` | Subtle triangle mesh | #0A1F3F bg, #00BCD4 at 10% opacity lines |
| `pattern-lines.svg` | Diagonal line hatch | #0A1F3F bg, #FDD835 at 8% opacity lines |
| `pattern-circles.svg` | Concentric circle fragments | #0A1F3F bg, #E91E63 at 6% opacity arcs |

### 3.3 Per-Category Placeholders

| Category | Placeholder File | Description |
|----------|-----------------|-------------|
| Hero | `placeholder-hero.webp` | Navy gradient (#0A1F3F → #0C2B5F) with dot mesh overlay |
| Services | `placeholder-service.svg` | Navy card with cyan geometric icon silhouette |
| Industries | `placeholder-industry.svg` | Navy card with white icon silhouette on cyan circle |
| Technology | `placeholder-tech.svg` | Dark navy gradient with equipment silhouette outline |
| About | `placeholder-about.svg` | Navy/dark gradient with abstract workspace outline |
| Testimonials | `placeholder-testimonial.svg` | White card with navy quote mark and dashed border |
| Offers | `placeholder-offer.svg` | Navy/white diagonal split with "Coming Soon" text zone |

### 3.4 `no-image.svg` Fallback

**Path:** `Frontend/public/images/placeholders/no-image.svg`

**Description:** A simple centered icon (a crossed-out image frame) with the Whites navy/white color scheme. Used as the final fallback in `<img>` `onError` handlers.

```
ViewBox: 0 0 200 200
Background: #F5F5F5
Icon: image icon in #0A1F3F with diagonal line through it
Text area: reserved below for optional label
```

### 3.5 Gradient Hero Placeholder

Used on the homepage before the hero photograph loads:

```
Frontend/public/images/placeholders/placeholder-hero-gradient.png
```

- 1920×1080px (scaled down for progressive loading)
- Background: linear-gradient(135deg, #0A1F3F 0%, #1A2F5F 50%, #0C2B5F 100%)
- Overlay: `pattern-dots.svg` tiled at 30% opacity
- Whites logo centred, tagline below in white

### 3.6 Implementation

In Next.js `<Image>` component:

```jsx
<Image
  src="/images/hero/hero-workspace.webp"
  placeholder="blur"
  blurDataURL="/images/placeholders/placeholder-hero-gradient.png"
  onError={(e) => {
    e.target.src = "/images/placeholders/no-image.svg";
  }}
/>
```

### 4.1 Specification

Used as the decorative background mesh on the homepage hero section (and optionally as a repeating texture on other navy sections).

**Path:** `Frontend/public/images/placeholders/patterns/hero-mesh.svg`

```svg
<svg xmlns="http://www.w3.org/2000/svg" width="800" height="600" viewBox="0 0 800 600">
  <rect width="800" height="600" fill="#0A1F3F" />
  <defs>
    <pattern id="mesh" x="0" y="0" width="40" height="40" patternUnits="userSpaceOnUse">

    <pattern id="mesh-large" x="0" y="0" width="120" height="120" patternUnits="userSpaceOnUse">
      <circle cx="60" cy="60" r="3" fill="rgba(255,255,255,0.04)" />
    </pattern>
    <radialGradient id="glow" cx="50%" cy="40%" r="60%">
      <stop offset="0%" stop-color="#00BCD4" stop-opacity="0.08" />
      <stop offset="50%" stop-color="#E91E63" stop-opacity="0.04" />
      <stop offset="100%" stop-color="#0A1F3F" stop-opacity="0" />

  <rect width="800" height="600" fill="url(#mesh-large)" />
  <rect width="800" height="600" fill="url(#glow)" />

```

**CSS implementation (preferred for performance):**

```css
.hero-background {
  background-color: #0A1F3F;
  background-image:
    radial-gradient(circle at 20px 20px, rgba(255,255,255,0.03) 1.5px, transparent 1.5px),

}
```

### 4.3 Usage

Applied as the base layer of the hero section. The hero photograph or abstract CMYK image appears as an overlay at 60% opacity on the right half, with text content on the left.

| Favicon 512×512 | PNG | 50KB | 85% |

JPEG fallback images: same dimensions as WebP, quality 80%, max size matching WebP targets.

### 7.2 Responsive Breakpoints & srcset

Three breakpoints per image asset:

| Breakpoint | Width | Usage |
|-----------|-------|-------|
| Small (mobile) | 480px | <640px viewports |
| Medium (tablet) | 768px | 640-1024px viewports |
| Large (desktop) | 1200px | >1024px viewports |

**Naming convention:**

```
Frontend/public/images/{category}/{description}-{size}.webp
```

Example:
```
Frontend/public/images/hero/hero-workspace-480.webp
Frontend/public/images/hero/hero-workspace-768.webp
Frontend/public/images/hero/hero-workspace-1200.webp
```

**Usage in Next.js:**

```jsx
// Hero (above the fold) — priority load, eager
```

### 7.3 CDN Strategy

All images served through Vercel's Edge Network / Next.js built-in image optimization.

**Remote patterns config (`next.config.js`):**

```js
module.exports = {
  images: {
    formats: ['image/webp', 'image/avif'],
    deviceSizes: [480, 768, 1200, 1920],
    imageSizes: [16, 32, 48, 96, 192, 256],
    minimumCacheTTL: 31536000, // 1 year
  },
};
```

Next.js `<Image>` component automatically:
- Serves WebP where browser supports it
- Generates AVIF where supported
- Resizes to requested dimensions
- Caches at edge for 1 year
- Applies quality optimizations
